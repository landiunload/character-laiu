# character laiu — анкета персонажа

Приложение для писателей: подробные карточки героев — внешность, характер,
мечты и страхи, связи, роль в сюжете и пять вопросов на глубину.

Работает целиком в браузере: без сервера, без регистрации, без отправки данных.
Написано на C# и работает через **Blazor WebAssembly**; первая версия проекта
была на чистом JavaScript и сохранена в истории репозитория.

## Возможности

- Полная анкета: 11 разделов, включая таблицу «Любит / Не любит» и авторские подсказки.
- Сколько угодно героев: создание, дублирование, удаление, список в боковой панели.
- Гибкая структура: поля и разделы можно добавлять, переименовывать, удалять
  и переставлять стрелками; значки разделов тоже редактируются.
- Все поля многострочные и растут под содержимое.
- Живой граф связей: узлы расходятся сами, их можно перетаскивать, связи бывают
  односторонними и двусторонними, тип связи и его цвет настраиваются заготовками.
- Автосохранение в localStorage браузера при каждом изменении.
- Экспорт анкеты в Markdown (готовый документ) и JSON (резервная копия), импорт JSON обратно.
- Печатная версия: кнопка «Печать» выводит чистый документ без панелей, кнопок и пустых полей.
- Светлая и тёмная темы, адаптивная вёрстка для телефона.

## Стек технологий

- **.NET 10 / Blazor WebAssembly** — весь код приложения на C#, выполняется в браузере
- **System.Text.Json с генератором сериализаторов** — хранение и экспорт без рефлексии
- **Canvas 2D** — граф связей рисуется и считается в JavaScript, вне цикла отрисовки Blazor
- **xUnit** — модульные тесты домена и экспортёров

## Архитектура

```
src/Charlaiu.Web/
├── Domain/        # Модель анкеты: герой, разделы, поля, связи, схема по умолчанию
├── Services/      # Хранилище (localStorage), экспортёры Markdown и JSON, скачивание файлов
├── Pages/         # Страница-редактор
└── wwwroot/       # Стили (включая печатную версию), граф связей и обёртка браузерных API
tests/Charlaiu.UnitTests/
```

Ключевые решения:

- **Хранилище за интерфейсом** `ICharacterProfileRepository`: реализация на
  localStorage заменяется на файловую или серверную без изменения компонентов.
- **Экспорт через общий контракт** `IQuestionnaireExporter`: новый формат — это
  новая реализация интерфейса, существующий код не меняется.
- **Схема анкеты — данные, а не код**: интерфейс рендерит любую структуру
  разделов и полей, поэтому пользователь свободно редактирует анкету под себя.
- Домен не зависит от Blazor и браузера, поэтому тестируется обычным xUnit.

Скорость:

- **Граф живёт в JavaScript** (`wwwroot/js/relationshipGraph.js`): физика и отрисовка
  идут через `requestAnimationFrame`, Blazor не трогает DOM на каждый кадр. Раскладка
  дуг считается один раз при смене данных, кадр только пересчитывает координаты,
  а в покое цикл засыпает и не потребляет ничего.
- **Каждая анкета под своим ключом** в localStorage, плюс ключ-указатель с их
  порядком. Правка одного героя стоит одной сериализации его анкеты и не зависит
  от того, сколько героев уже создано. Пока всё лежало одним массивом, нажатие
  клавиши при тридцати заполненных анкетах занимало 0,56 с; сейчас — 20 мс,
  столько же, сколько при одной. Сохранения первой версии формата переносятся
  автоматически при первом запуске.
- **Сериализация без рефлексии**: анкеты уходят в localStorage после каждого нажатия
  клавиши, поэтому сериализаторы генерируются на этапе сборки (`CharlaiuJsonContext`).
- **Отказ от данных ICU** (`InvariantGlobalization`): минус несколько мегабайт
  в публикации и более быстрый холодный старт.

## Безопасность

- Данные не покидают устройство: нет сервера, нет аналитики, нет внешних запросов.
- Строгая Content Security Policy: скрипты и стили только со своего источника,
  встраивание сайта в iframe запрещено.
- Пользовательский ввод выводится через биндинги Blazor, которые всегда экранируют
  текст, — разметка из данных не выполняется (защита от XSS).
- Импорт принимает только корректный JSON ограниченного размера.

## Запуск

Нужен установленный [.NET SDK 10](https://dotnet.microsoft.com/download/dotnet/10.0):

```bash
dotnet run --project src/Charlaiu.Web
```

Адрес будет в выводе команды (обычно http://localhost:5000).

## Тесты

```bash
dotnet test
```
