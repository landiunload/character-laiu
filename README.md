# 📖 «Где он лёг» — анкета персонажа

Минималистичное веб-приложение для ведения карточек героев: внешность, характер,
мечты и страхи, связи, роль в истории и пять вопросов на глубину.

Без регистрации, без сервера, без зависимостей — чистые HTML, CSS и JavaScript.

## Возможности

- ✍️ **Полная анкета** по структуре «Где он лёг»: 11 разделов, включая таблицу
  «Любит / Не любит» и авторские подсказки.
- 👥 **Сколько угодно героев** — список в боковой панели, создание, копирование, удаление.
- 🧩 **Гибкая структура** — любое поле и любой раздел можно добавить, переименовать
  или убрать; подписи полей тоже редактируются.
- 💾 **Автосохранение** в `localStorage` браузера — ничего никуда не отправляется.
- 📤 **Экспорт** анкеты в JSON (резервная копия) и Markdown (готовый документ),
  экспорт всех героев одним файлом и **импорт** обратно.
- 🌙 **Светлая и тёмная темы**, адаптивная вёрстка для телефона.

## Запуск

Нужен любой статический сервер (ES-модули не работают с `file://`):

```bash
# Python
python3 -m http.server 8000

# либо Node.js
npx serve .
```

Затем откройте <http://localhost:8000>.

## Безопасность

- Нет бэкенда и базы данных — данные живут только в браузере пользователя.
- Строгая Content Security Policy: загрузка скриптов и стилей только со своего
  источника, без inline-кода и внешних запросов.
- Пользовательский текст попадает в страницу только через `textContent` и
  `value` — разметка из данных никогда не выполняется (защита от XSS).
- Импортируемый JSON проверяется и пересобирается поле за полем: лишние ключи
  отбрасываются, значения приводятся к строкам и ограничиваются по длине.

## Архитектура

Модули разделены по ответственности (SOLID); комментарии и логи — на русском.

```
index.html        — каркас страницы
css/style.css     — оформление (CSS-переменные, светлая/тёмная темы)
js/
  app.js          — контроллер: связывает модули, обрабатывает действия
  schema.js       — схема анкеты по умолчанию и фабрики разделов/полей
  repository.js   — хранилище (localStorage), легко заменить на сервер
  render.js       — отрисовка интерфейса, безопасная работа с DOM
  transfer.js     — экспорт/импорт JSON и Markdown, проверка входных данных
  utils.js        — мелкие помощники (id, debounce, скачивание файлов)
```

- **S** — каждый модуль делает одно дело: хранение, отрисовка, экспорт.
- **O** — новая структура анкеты добавляется данными в `schema.js`, без правок логики.
- **L** — репозиторий принимает любое хранилище с интерфейсом `localStorage`.
- **I/D** — отрисовка сообщает об изменениях через переданные обработчики
  и ничего не знает о хранилище; контроллер связывает их между собой.
